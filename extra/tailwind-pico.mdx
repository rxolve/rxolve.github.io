# Next.js 14에서 Pico CSS와 Tailwind CSS 동시 적용하기

이 가이드에서는 Next.js 14 프로젝트에 Pico CSS와 Tailwind CSS를 함께 적용하는 방법을 단계별로 설명합니다.

## 1. Next.js 프로젝트 생성

먼저, 새로운 Next.js 14 프로젝트를 생성합니다:

```bash
npx create-next-app@latest my-next-pico-tailwind-app
cd my-next-pico-tailwind-app
```

프로젝트 생성 시 Tailwind CSS 설치 여부를 묻는 질문에 'Yes'라고 답하세요.

## 2. Pico CSS 설치

Next.js 프로젝트에 Pico CSS를 설치합니다:

```bash
npm install @picocss/pico
```

## 3. Tailwind CSS 설정 확인

`tailwind.config.js` 파일이 다음과 같이 설정되어 있는지 확인합니다:

```javascript
/** @type {import('tailwindcss').Config} */
module.exports = {
  content: [
    "./pages/**/*.{js,ts,jsx,tsx,mdx}",
    "./components/**/*.{js,ts,jsx,tsx,mdx}",
    "./app/**/*.{js,ts,jsx,tsx,mdx}",
  ],
  theme: {
    extend: {},
  },
  plugins: [],
};
```

## 4. 전역 스타일 설정

`app/globals.css` 파일을 열고 다음과 같이 수정합니다:

```css
@import "@picocss/pico/css/pico.min.css";

@tailwind base;
@tailwind components;
@tailwind utilities;

/* 추가적인 커스텀 스타일 */
```

이렇게 하면 Pico CSS의 스타일을 먼저 불러온 다음, Tailwind의 스타일을 적용합니다.

## 5. Layout 컴포넌트 수정

`app/layout.js` 파일을 열고 다음과 같이 수정합니다:

```javascript
import "./globals.css";

export const metadata = {
  title: "Next.js with Pico and Tailwind",
  description: "A Next.js app using both Pico CSS and Tailwind CSS",
};

export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  );
}
```

## 6. 페이지 컴포넌트 생성

`app/page.js` 파일을 다음과 같이 수정하여 Pico CSS와 Tailwind CSS를 모두 사용하는 예제를 만듭니다:

```jsx
export default function Home() {
  return (
    <main className="container">
      <h1 className="text-3xl font-bold text-center my-8">
        Next.js 14 + Pico CSS + Tailwind CSS
      </h1>
      <p className="text-gray-600 text-center">
        이 페이지는 Pico CSS와 Tailwind CSS를 함께 사용합니다.
      </p>
      <div className="grid">
        <button className="bg-blue-500 hover:bg-blue-700 text-white font-bold py-2 px-4 rounded">
          Tailwind 스타일 버튼
        </button>
        <button>Pico 기본 버튼</button>
      </div>
    </main>
  );
}
```

## 7. 개발 서버 실행

다음 명령어로 개발 서버를 실행합니다:

```bash
npm run dev
```

이제 `http://localhost:3000`에서 Pico CSS와 Tailwind CSS가 함께 적용된 페이지를 확인할 수 있습니다.

## 주의사항

1. **스타일 충돌**: Pico CSS와 Tailwind CSS 사이에 스타일 충돌이 발생할 수 있습니다. 이런 경우 Tailwind의 `@layer` 지시어를 사용하여 우선순위를 조정하거나, 더 구체적인 선택자를 사용해야 할 수 있습니다.

2. **성능**: 두 CSS 프레임워크를 함께 사용하면 CSS 파일 크기가 커질 수 있습니다. 프로덕션 빌드 시 Next.js의 기본 최적화를 통해 일부 해결되지만, 추가적인 최적화가 필요할 수 있습니다.

3. **일관성**: Pico CSS와 Tailwind CSS는 디자인 철학이 다르기 때문에, 일관된 디자인 시스템을 유지하기 어려울 수 있습니다. 프로젝트의 요구사항을 잘 고려하여 사용해야 합니다.

4. **유지보수**: 두 프레임워크를 동시에 사용하면 코드베이스가 복잡해질 수 있으며, 새로운 팀 멤버들이 프로젝트에 적응하는 데 시간이 더 걸릴 수 있습니다.

## 결론

Next.js 14에서 Pico CSS와 Tailwind CSS를 함께 사용하면 각 프레임워크의 장점을 활용할 수 있습니다. Pico CSS의 기본 스타일링과 반응형 디자인을 기반으로 하면서, Tailwind CSS의 유틸리티 클래스로 세밀한 커스터마이징을 할 수 있습니다.

하지만 이 접근 방식이 모든 프로젝트에 적합한 것은 아닙니다. 프로젝트의 요구사항, 팀의 선호도, 그리고 유지보수 용이성을 고려하여 결정해야 합니다. 대부분의 경우, 두 프레임워크 중 하나만 선택하여 사용하는 것이 더 간단하고 효율적일 수 있습니다.
