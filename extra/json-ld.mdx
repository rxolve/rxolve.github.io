#### JSON-LD 란?

- JSON-LD(JavaScript Object Notation for Linked Data)는 구조화된 데이터를 표현하기 위한 경량 형식입니다.
- 이 형식은 검색 엔진이 웹 페이지의 콘텐츠를 더 정확하게 이해하고 해석할 수 있게 해줍니다.
- Google, Bing 등 주요 검색 엔진에서 권장하는 구조화된 데이터 형식입니다.

#### JSON-LD의 역사

시맨틱 웹의 꿈
JSON-LD의 탄생을 이해하려면 먼저 시맨틱 웹(Semantic Web)의 개념을 알아야 합니다.

1989년: Tim Berners-Lee가 월드 와이드 웹(WWW)을 발명
1998년: Tim Berners-Lee가 "시맨틱 웹"의 개념 제안

목표: 기계가 웹의 데이터를 이해하고 처리할 수 있는 환경 구축

시맨틱 웹의 비전은 웹 페이지의 내용을 단순한 텍스트가 아닌 의미 있는 데이터로 표현하는 것이었습니다. 2. 초기 시맨틱 웹 기술

RDF (Resource Description Framework): 1999년 도입
OWL (Web Ontology Language): 2004년 도입

이러한 기술들은 강력했지만, 복잡성으로 인해 널리 채택되지 못했습니다.

3. JSON-LD의 탄생

2010년: Manu Sporny와 Dave Longley가 JSON-LD 개념 제안
목표: 개발자 친화적인 방식으로 링크드 데이터(Linked Data) 구현

JSON-LD는 다음과 같은 필요성에 의해 탄생했습니다:

단순성: 기존 JSON 구문을 그대로 사용하여 학습 곡선 낮춤
호환성: 기존 JSON 기반 시스템과의 높은 호환성
유연성: 복잡한 데이터 구조를 표현할 수 있는 능력

4. JSON-LD의 발전

2012년: JSON-LD 1.0 초안 발표
2014년: W3C 권장 사항으로 승인
2020년: JSON-LD 1.1 버전 출시

#### JSON-LD 구조

JSON-LD 문서는 항상 JSON 형식이며, 다음과 같은 기본 구조를 갖습니다.

```json
{
  "@context": "https://schema.org",
  "@type": "Type",
  "property1": "value1",
  "property2": "value2",
  ...
}
```

- @context: JSON-LD 문서의 컨텍스트를 정의합니다. 보통 "https://schema.org" 를 사용합니다.
- @type: 설명하는 엔티티의 유형을 지정합니다.

#### 중첩과 참조

JSON-LD는 중첩 구조를 지원합니다.

```json
{
  "@context": "https://schema.org",
  "@type": "Article",
  "author": {
    "@type": "Person",
    "name": "John Doe",
    "sameAs": "https://example.com/johndoe"
  },
  "publisher": {
    "@type": "Organization",
    "name": "Tech Blog Inc.",
    "logo": {
      "@type": "ImageObject",
      "url": "https://example.com/logo.png"
    }
  }
}
```

또한, @id를 사용하여 다른 엔티티를 참조할 수 있습니다.

```json
{
  "@context": "https://schema.org",
  "@graph": [
    {
      "@type": "Person",
      "@id": "#jane",
      "name": "Jane Doe"
    },
    {
      "@type": "Article",
      "author": { "@id": "#jane" }
    }
  ]
}
```

### Next.js에서 JSON-LD 사용하기

- Next.js 13.2 이상 버전에서 generateMetadata 활용 해서 메타데이터와 JSON-LD를 한 곳에서 관리할 수 있습니다.
  - 이외 `<script>`태그 안에 직접 삽입, react-schemaorg 라이브러리 사용 등의 방식이 있습니다.

```tsx
import { Metadata } from "next";

interface BlogPostProps {
  params: { id: string };
}

export async function generateMetadata({
  params,
}: BlogPostProps): Promise<Metadata> {
  const post = await getPostData(params.id);

  return {
    title: post.title,
    description: post.summary,
    openGraph: {
      title: post.title,
      description: post.summary,
      type: "article",
      publishedTime: post.date,
      authors: [post.author],
    },
    other: {
      "application/ld+json": JSON.stringify({
        "@context": "https://schema.org",
        "@type": "BlogPosting",
        headline: post.title,
        datePublished: post.date,
        author: {
          "@type": "Person",
          name: post.author,
        },
        description: post.summary,
      }),
    },
  };
}

export default function BlogPost({ params }: BlogPostProps) {
  // 컴포넌트 렌더링 로직
}
```

### 정리

- JSON-LD는 Next.js 애플리케이션의 SEO를 크게 개선할 수 있는 강력한 도구입니다. 적절히 구현하면 검색 엔진이 여러분의 콘텐츠를 더 잘 이해하고, 검색 결과에서 더 풍부한 정보를 표시할 수 있게 됩니다.
- JSON-LD를 단순히 구현하는 것을 넘어 최적화하고 성능을 고려하는 것이 중요합니다. 이를 통해 기술적 SEO의 효과를 극대화할 수 있습니다.
