# this와 실행 컨텍스트

자바스크립트를 배우는 주니어 개발자들이 가장 헷갈려하는 개념 중 하나가 바로 'this'와 실행 컨텍스트입니다. 이 포스트에서는 이 두 개념에 대해 자세히 알아보겠습니다.

## 1. 실행 컨텍스트란?

실행 컨텍스트는 자바스크립트 코드가 실행되는 환경을 말합니다. 코드가 실행될 때 자바스크립트 엔진은 실행 컨텍스트를 생성하고, 이 안에서 변수, 함수 선언, this 바인딩 등의 정보를 관리합니다.

실행 컨텍스트는 크게 세 가지로 나눌 수 있습니다:

- **전역 실행 컨텍스트**: 코드가 처음 실행될 때 생성되는 기본 컨텍스트
- **함수 실행 컨텍스트**: 함수가 호출될 때마다 생성되는 컨텍스트
- **eval 실행 컨텍스트**: eval 함수 내에서 실행되는 코드의 컨텍스트

## 2. this의 존재 이유

'this'는 자바스크립트에서 현재 실행 컨텍스트를 참조하는 특별한 키워드입니다. 'this'의 주요 존재 이유는 다음과 같습니다:

### a) 객체 지향 프로그래밍 지원

'this'를 통해 메서드가 자신이 속한 객체를 참조할 수 있습니다. 이로 인해 객체의 속성과 메서드를 더 유연하게 사용할 수 있습니다.

### b) 코드 재사용성 향상

동일한 함수를 여러 객체에서 재사용할 수 있게 해줍니다. 'this'는 함수가 어떤 객체에 바인딩되었는지에 따라 동적으로 결정되므로, 하나의 함수를 여러 객체에서 메서드로 사용할 수 있습니다.

### c) 이벤트 처리의 편의성

이벤트 핸들러에서 'this'는 일반적으로 이벤트가 발생한 요소를 가리킵니다. 이를 통해 이벤트 처리 로직을 더 직관적으로 작성할 수 있습니다.

## 3. this와 실행 컨텍스트의 관계

'this'의 값은 실행 컨텍스트에 따라 동적으로 결정됩니다. 즉, 같은 함수라도 어떻게 호출되었는지, 어떤 컨텍스트에서 실행되는지에 따라 'this'가 가리키는 대상이 달라집니다.

예를 들어:

- 전역 스코프에서 'this'는 전역 객체(브라우저에서는 window)를 가리킵니다.
- 메서드로 호출된 함수에서 'this'는 해당 메서드를 소유한 객체를 가리킵니다.
- 생성자 함수에서 'this'는 새로 생성되는 인스턴스를 가리킵니다.

이해를 돕기 위해 간단한 예제를 보여드리겠습니다:

```javascript
const person = {
  name: "Alice",
  greet: function () {
    console.log(`안녕하세요, 제 이름은 ${this.name}입니다.`);
  },
};

person.greet(); // 출력: 안녕하세요, 제 이름은 Alice입니다.

const greetFunction = person.greet;
greetFunction(); // 출력: 안녕하세요, 제 이름은 undefined입니다.
```

이 예제에서 `person.greet()`를 호출할 때, 'this'는 'person' 객체를 가리킵니다. 하지만 `greetFunction()`으로 호출할 때는 'this'가 전역 객체(또는 strict 모드에서는 undefined)를 가리키게 되어 다른 결과가 나옵니다.

## 결론

'this'와 실행 컨텍스트는 자바스크립트의 동작을 이해하는 데 중요한 개념입니다. 이들을 잘 이해하면 더 유연하고 강력한 코드를 작성할 수 있습니다. 계속해서 실습하고 다양한 상황에서 'this'의 동작을 관찰하면서 이해도를 높여가시기 바랍니다.

> Context is king. 'this'는 그 말을 가장 잘 보여주는 자바스크립트의 특징입니다.
