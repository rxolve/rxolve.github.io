# DP를 이용한 최대 직사각형 합 문제의 효율적 해결

2D 배열에서 최대 합을 가진 직사각형을 찾는 문제는 카데인 알고리즘을 2D로 확장하여 해결할 수 있습니다. 하지만 이 방법의 시간 복잡도는 O(n^3)입니다. 여기서는 동적 프로그래밍(DP)을 사용하여 이 문제를 더 효율적으로 해결하는 방법을 알아보겠습니다.

## 문제 정의

2D 정수 배열이 주어졌을 때, 합이 최대가 되는 직사각형 영역의 합을 찾는 문제입니다.

## DP를 이용한 해결 방법

이 문제를 해결하기 위해 우리는 다음과 같은 접근 방식을 사용할 것입니다:

1. 각 열에 대해 누적 합을 계산합니다.
2. 모든 가능한 행 쌍에 대해 1D 최대 부분 합 문제로 변환합니다.
3. 변환된 1D 문제를 O(n) 시간에 해결합니다.

## 구현

다음은 TypeScript로 구현한 코드입니다:

```typescript
function maxRectangleSum(matrix: number[][]): number {
  if (!matrix || matrix.length === 0 || matrix[0].length === 0) {
    return 0;
  }

  const rows = matrix.length;
  const cols = matrix[0].length;
  let maxSum = Number.NEGATIVE_INFINITY;

  // 누적 합 배열 생성
  const prefixSum = Array.from({ length: rows + 1 }, () =>
    new Array(cols).fill(0)
  );

  // 누적 합 계산
  for (let i = 1; i <= rows; i++) {
    for (let j = 0; j < cols; j++) {
      prefixSum[i][j] = prefixSum[i - 1][j] + matrix[i - 1][j];
    }
  }

  // 모든 가능한 행 쌍에 대해 반복
  for (let top = 0; top < rows; top++) {
    for (let bottom = top; bottom < rows; bottom++) {
      let currentSum = 0;
      let maxEndingHere = 0;

      // 1D 최대 부분 합 문제 해결
      for (let j = 0; j < cols; j++) {
        currentSum = prefixSum[bottom + 1][j] - prefixSum[top][j];
        maxEndingHere = Math.max(currentSum, maxEndingHere + currentSum);
        maxSum = Math.max(maxSum, maxEndingHere);
      }
    }
  }

  return maxSum;
}

// 테스트
const matrix = [
  [1, 2, -1, -4, -20],
  [-8, -3, 4, 2, 1],
  [3, 8, 10, 1, 3],
  [-4, -1, 1, 7, -6],
];

console.log(maxRectangleSum(matrix)); // 출력: 29
```

## 알고리즘 설명

1. **누적 합 계산**: 먼저 각 열에 대한 누적 합을 계산합니다. 이를 통해 임의의 직사각형 영역의 합을 O(1) 시간에 계산할 수 있습니다.

2. **행 쌍 선택**: 모든 가능한 상단 행(top)과 하단 행(bottom) 쌍에 대해 반복합니다.

3. **1D 문제로 변환**: 선택된 행 사이의 영역을 1D 배열로 변환합니다. 각 원소는 해당 열의 누적 합 차이입니다.

4. **1D 최대 부분 합 찾기**: 변환된 1D 배열에서 최대 부분 합을 찾습니다. 이는 카데인 알고리즘과 유사한 방식으로 O(n) 시간에 해결됩니다.

5. **최대값 갱신**: 찾은 최대 부분 합이 전체 최대값보다 크다면 갱신합니다.

## 시간 복잡도 분석

- 누적 합 계산: O(rows \* cols)
- 행 쌍 선택: O(rows^2)
- 각 행 쌍에 대한 1D 최대 부분 합 찾기: O(cols)

따라서 전체 시간 복잡도는 O(rows^2 _ cols)입니다. 이는 이전의 O(rows^3 _ cols) 접근 방식보다 훨씬 효율적입니다.

## 공간 복잡도

이 알고리즘은 누적 합을 저장하기 위해 추가적인 O(rows \* cols) 공간을 사용합니다.

## 결론

동적 프로그래밍을 활용한 이 접근 방식은 2D 카데인 알고리즘의 시간 복잡도를 O(rows^3 _ cols)에서 O(rows^2 _ cols)로 개선합니다. 이는 특히 열의 수가 많은 큰 행렬에서 상당한 성능 향상을 제공합니다.

이 최적화된 알고리즘은 이미지 처리, 데이터 분석, 금융 모델링 등 대규모 2D 데이터를 다루는 다양한 실제 응용 프로그램에서 유용하게 사용될 수 있습니다.
