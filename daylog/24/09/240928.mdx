타입스크립트와 코틀린은 각각 자바스크립트와 자바의 한계를 극복하기 위해 만들어진 언어입니다.

(자바스크립트도 이름만 자바에서 영향을 받긴 했네요)

코틀린을 몇 년만에 들여다보며 두 언어가 놀란만큼 닮았다는 생각이 들었습니다.

## 정적 타입 시스템

#### 타입스크립트

타입스크립트는 자바스크립트의 자유로운 동적 타이핑이 가져오는 불안정성을 해결하기 위해 정적 타입 시스템을 도입했습니다.

기존 자바스크립트 코드에 점진적으로 타입을 추가할 수 있어, 대규모 애플리케이션에서 코드의 안전성과 유지 보수성을 높일 수 있습니다.

```javascript
// JavaScript: 동적 타입의 문제점
function add(a, b) {
  return a + b; // a와 b의 타입이 명확하지 않음
}

console.log(add(5, "5")); // '55'
```

```typescript
// TypeScript: 정적 타입의 안전성
function add(a: number, b: number): number {
  return a + b; // a와 b는 숫자 타입으로 고정됨
}

console.log(add(5, 5)); // 10
```

타입스크립트는 위와 같이 코드 작성 시 타입 오류를 사전에 방지하여 예측 가능한 동작을 보장합니다.

#### 코틀린

코틀린 역시 자바의 타입 시스템을 개선하여 컴파일 타임에 오류를 사전에 검출하고, 더 안전한 코드를 작성할 수 있도록 합니다.

특히 Null Safety는 코틀린의 가장 큰 장점 중 하나로, 널 참조로 인한 런타임 오류를 효과적으로 방지합니다.

```java
// Java: NullPointerException 발생 가능
String name = null;
System.out.println(name.length()); // NPE 발생

// Kotlin: Null Safety를 통해 안전한 코드 작성
val name: String? = null
println(name?.length) // 안전한 호출
```

타입스크립트와 코틀린은 각각의 플랫폼에서 발전된 타입 시스템으로 코드의 안전성을 극대화하고 있습니다.

## 함수형 프로그래밍

#### 타입스크립트

타입스크립트는 자바스크립트에서 파생되었기에 함수형 프로그래밍을 하기 편리한 문법을 제공합니다.

고차 함수, 람다 표현식, 클로저 등 다양한 함수형 프로그래밍 요소를 활용하여 코드의 가독성을 높이고, 사이드이펙트를 줄일 수 있습니다.

```typescript
// TypeScript: 고차 함수와 람다 표현식
const numbers = [1, 2, 3, 4];
const doubled = numbers.map((n) => n * 2);
console.log(doubled); // [2, 4, 6, 8]
```

#### 코틀린

코틀린은 자바보다 더욱 강력한 함수형 프로그래밍 기능을 제공합니다.

코틀린의 람다 표현식과 고차 함수는 코드의 간결함을 극대화하고, 불변성을 강조하여 안전한 병렬 처리를 가능하게 합니다.

또한, `with`, `apply`와 같은 스코프 함수는 객체 초기화나 설정을 보다 직관적으로 작성할 수 있게 해줍니다.

```kotlin
// Kotlin: 람다 표현식과 스코프 함수
val numbers = listOf(1, 2, 3, 4)
val doubled = numbers.map { it * 2 }
println(doubled) // [2, 4, 6, 8]

data class User(var name: String, var age: Int)

val user = User("John", 30).apply {
    age += 1 // 스코프 함수로 인한 간결한 코드
}
println(user) // User(name=John, age=31)
```

타입스크립트와 코틀린 모두 함수형 프로그래밍의 장점을 살리면서 개발자 경험을 크게 개선했습니다.

## 언어의 유연성과 확장성

#### 타입스크립트

타입스크립트의 가장 큰 장점 중 하나는 기존 자바스크립트 코드와의 호환성입니다.

기존 프로젝트에 타입스크립트를 점진적으로 도입하거나, 필요에 따라 타입을 적용하지 않는 등 유연한 사용이 가능합니다.

이를 통해 팀 전체가 점진적으로 타입스크립트를 수용하면서, 기존 자바스크립트 생태계를 그대로 활용할 수 있습니다.

```typescript
// JavaScript 코드도 그대로 사용 가능
const message = "Hello, world!";
console.log(message);
```

#### 코틀린

코틀린은 자바와 100% 호환성을 보장합니다.

기존 자바 코드를 그대로 사용하거나, 자바 라이브러리를 코틀린에서 문제 없이 사용할 수 있습니다.

이로 인해 기존 자바 기반 프로젝트에 코틀린을 도입하거나, 자바 개발자들이 코틀린으로 전환하는 데 진입 장벽이 낮습니다.

또한 코틀린의 확장 함수는 기존 자바 API를 더욱 편리하게 사용할 수 있도록 해줍니다.

```kotlin
// Java 코드와의 완벽한 호환
val list = ArrayList<String>()
list.add("Hello")
list.add("Kotlin")

// Kotlin의 확장 함수로 간결한 코드 작성
list.forEach { println(it) }
```

타입스크립트와 코틀린 모두 기존 생태계와의 호환성을 강조하여, 새로운 언어 도입의 부담을 줄이고 개발자 경험을 개선합니다.

## 4. 멀티플랫폼 지원

#### 타입스크립트

타입스크립트는 자바스크립트와 마찬가지로 브라우저, Node.js, 모바일, 데스크탑 등 다양한 환경에서 사용할 수 있습니다.

React Native와 같은 프레임워크를 사용하면 타입스크립트를 활용한 모바일 애플리케이션 개발도 가능합니다.

### 코틀린

코틀린은 JVM 기반 애플리케이션뿐만 아니라, Kotlin/JS와 Kotlin/Native를 통해 자바스크립트와 네이티브 플랫폼으로도 컴파일할 수 있습니다.

이를 통해 안드로이드, iOS, 웹 애플리케이션 등 여러 플랫폼에서 공통 코드를 공유하며 개발할 수 있습니다.

## 정리하며

물론 타입스크립트과 코틀린은 태생적으로 서로 다른 패러다임을 가지고 있습니다.

그럼에도 불구하고 현대적 프로그래밍을 위한 다양한 기능을 제공하며, 서로 닮아있는 모습이 흥미로웠습니다.

마치 NestJS와 Spring Boot가 서로 닮아있는 것처럼, 타입스크립트와 코틀린도 서로 다른 플랫폼에서 비슷한 개발 경험을 제공하고 있습니다.
