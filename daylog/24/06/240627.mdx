> 정리하며 책의 문장을 요약해서 옮깁니다.

- 코드의 어떤 표현식들은 계속 성장합니다.
  - 그러다가 돋보기를 동원해야만 무슨 내용인지 알게 되는 지경에 이릅니다.
- 복잡한 코드의 표현식을 이해했다면 전체에서 일부 표현식을 추출한 후, 변수 이름을 만들어 할당해 보세요.
- Before
  ```
  return new Point(
    ...긴 표현식...,
    ...다른 긴 표현식...
  )
  ```
- After

  ```
  x = ...긴 표현식...
  y = ...다른 긴 표현식...
  return new Point(x, y)
  ```

- 이 정리법은 여러분이 힘들게 파악한 내용을 다시 코드에 넣는 것입니다.
- 이렇게 설명하는 변수를 적용하면 표현식과 분리가 됩니다.
  - 다음번 코드를 변경할 때, 재빨리 찾아낼 수 있습니다.
- 언제나 그렇듯이, 코드 정리에 대한 커밋과 동작 변경에 대한 커밋은 분리해야 합니다.
