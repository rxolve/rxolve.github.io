---
title: "git rebase: 커밋 날짜 변경하기"
date: "240630"
tags: ["Git"]
---

- 주말에 여행을 다녀왔습니다. 어제 포스팅을 올리지 못하고 이제야 느즈막히 맥북을 열었습니다.
- 빈 잔디가 허전해보여서 git rebase 로 커밋 날짜를 변경하는 방법을 정리해봅니다.

1. 날짜를 변경할 커밋 올리기
   - 미리 작성해놓은 포스팅을 커밋했습니다.
     - [240629: Tidy First? 9. 설명하는 상수](../tag/TidyFirst/240629)
2. rebase 시작
   - 변경하려는 커밋이 가장 최근 커밋이라서 HEAD~1로 지정합니다.
   ```bash
   git rebase -i HEAD~1
   ```
   - 여기서 -i는 interactive 모드를 의미합니다. 텍스트 편집기가 열리면서 rebase할 커밋 리스트가 표시됩니다.
3. 커밋 수정
   - 변경하려는 커밋 앞에 edit를 입력하고 저장 후 편집기를 닫습니다.
     - vi에서 x로 글자를 지우고 i로 입력모드로 전환한 후 edit를 입력하고 :wq로 저장합니다.
   - Before:
     ```bash
        pick <커밋 해시> <커밋 메시지>
     ```
   - After:
     ```bash
        edit <커밋 해시> <커밋 메시지>
     ```
4. 커밋 날짜 변경
   - 이제 rebase가 일시 중지되고, 변경하려는 커밋으로 이동됩니다.
   ```bash
   GIT_COMMITTER_DATE="2024-06-29T12:00:00" git commit --amend --no-edit --date "2024-06-29T12:00:00"
   ```
5. rebase 계속
   - 변경이 완료되면 rebase 를 계속 진행합니다.
   ```bash
   git rebase --continue
   ```
   - 여기서 충돌이 발생할 경우, 충돌을 해결하고 'git add 파일'을 사용하여 변경 사항을 스테이징한 후 다시 'git rebase --continue'를 실행합니다.
6. 변경 사항 푸시
   - 변경된 커밋을 원격 저장소에 푸시합니다. 이때 푸시에 실패하면 --force 옵션을 사용하여 강제로 푸시합니다.
   ```bash
    git push --force
   ```
7. 변경 사항 확인
   - 이 과정을 통해 커밋 날짜를 변경할 수 있었습니다.
   - 앞으로는 여행을 갈 때에 맥북을 들고 다닐 예정입니다.
