> 김철수님의 `개발자의 글쓰기`를 읽고 정리한 글입니다.

비즈니스 문서에는 문단과 문단 사이에 위계가 있어야 합니다.

```
위계 = 위치 + 계층
```

개발 툴은 2차원인 위치는 맞출 수 있지만 3차원인 계층을 맞출 수는 없습니다.

```js
var a;
var b;
if (a > b) {
  return a;
} else {
  return b;
}
```

개발자가 쓴 문서는 위치는 맞추지만 계층은 거의 표현하지 않는 경향이 있습니다.

그래서 기획자나 관리자가 보면 마치 메모장으로 문서를 작성한 느낌을 받을 수 있습니다.

사람은 계층이 표현되지 않은 문서를 보면 체계를 잡아내기가 무척 어렵습니다. 컴퓨터가 아니기 때문입니다.

계층을 표현하세요. 읽는 사람의 시간을 아껴주세요.

글을 쓸 때는 위계가 잘 드러나야 합니다.

## 위치만 맞춘 예시

#### 서비스 업그레이드 결과

- 이번 대규모 업그레이드로 특징은 3가지입니다.
  - 완전히 새로운 기능이 추가되었습니다.
  - 버그 없는 안전성을 확보했습니다.
  - 인공지능으로 콜센터를 대체했습니다.
- 업그레이드 과정은 3단계로...
