## 튜링 완전(Turing-completeness)이란?

튜링 완전은 어떤 프로그래밍 언어나 추상 기계가 튜링 기계와 동등한 계산 능력을 가지고 있다는 것을 의미합니다.

다시 말해, 튜링 기계로 계산할 수 있는 모든 것을 계산할 수 있는 시스템을 튜링 완전하다고 합니다.

## 역사적 배경

이 개념은 이름에서 짐작 가능한대로 앨런 튜링이 1936년에 발표한 논문 "On Computable Numbers, with an Application to the Entscheidungsproblem"에서 처음 소개되었습니다.

튜링은 이 논문에서 계산 가능성의 형식적 정의를 제시했고, 이는 현대 컴퓨터 과학의 기초가 되었습니다.

## 튜링 완전의 의미

튜링 완전 시스템은 다음과 같은 특징을 가집니다.

- 무한한 메모리: 이론적으로 무한한 양의 메모리를 사용할 수 있어야 합니다. 실제 컴퓨터는 물리적으로 한계가 있지만, 논리적으로 무한한 메모리를 사용할 수 있다고 가정합니다.

- 조건 분기: 프로그램이 특정 조건에 따라 다른 명령을 실행할 수 있어야 합니다. 예를 들어, if-else 문이 이에 해당합니다.

- 반복: 특정 조건이 만족될 때까지 명령을 반복 실행할 수 있어야 합니다. 예를 들어, while 또는 for 루프가 이에 해당합니다.

## 현대 프로그래밍에서의 튜링 완전

대부분의 현대 프로그래밍 언어들은 튜링 완전합니다. 하지만 몇 가지 재밌는 예외가 있습니다.

- SQL: 순수한 SQL은 튜링 완전하지 못하지만, PostgreSQL 같은 확장은 튜링 완전합니다.
  ```sql
  WITH RECURSIVE fibonacci(n, a, b) AS (
      SELECT 1, 0, 1
      UNION ALL
      SELECT n + 1, b, a + b
      FROM fibonacci
      WHERE n < 5
  )
  SELECT b
  FROM fibonacci
  WHERE n = 5;
  ```
- HTML+CSS: CSS3와 HTML5는 함께 사용될 때 놀랍게도 튜링 완전합니다.
  - [https://codepen.io/vlearnweb/pen/jGyKBm](https://codepen.io/vlearnweb/pen/jGyKBm)
- 타입스크립트의 타입 시스템: TS 타입 시스템만으로 프로그램을 작성할 수 있습니다.

  ```ts
  type Fibonacci<T extends number> = T extends 0 | 1
    ? 1
    : T extends 2
    ? 1
    : [...Fibonacci<Subtract<T, 1>>, ...Fibonacci<Subtract<T, 2>>]["length"];

  type Result = Fibonacci<5>; // 5
  ```

## 마치며

튜링 완전은 컴퓨터 과학의 근본적인 개념이며, 현대 프로그래밍 언어와 시스템의 능력을 이해하는 데 중요합니다.

시스템이 튜링 완전한지 아는 것은 그 시스템의 능력과 한계를 이해하는 데 도움을 줍니다.
