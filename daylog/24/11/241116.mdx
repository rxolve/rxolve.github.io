## 왜 PR의 크기가 중요한가?

- 리뷰 효율성:  
  PR이 너무 크면 리뷰어는 한 번에 너무 많은 변경 사항을 파악해야 합니다.  
  이로 인해 집중력이 떨어지고, 결국 리뷰 품질이 저하될 수 있습니다.  
  반면, 적절한 크기의 PR은 리뷰어가 1시간 이내에 전체를 검토할 수 있게 해주어, 세세한 피드백이 가능해집니다.

- 협업과 커뮤니케이션:  
  한 PR에 하나의 기능이나 이슈를 담으면, 코드의 목적과 의도가 명확해지고 팀원 간 소통도 원활해집니다.  
  이는 단순히 코드 변경을 넘어서, 프로젝트 전반의 방향성 파악에도 큰 도움이 됩니다.

## 작은 PR과 큰 PR의 장단점

### 작은 PR의 장점

- 리뷰가 용이: 변경 사항이 작을수록 리뷰어는 쉽게 코드를 이해하고 빠르게 피드백을 줄 수 있습니다.
- 빠른 피드백: 짧은 PR은 빠른 리뷰와 merge를 가능하게 하여, 이후 작업에 신속하게 반영할 수 있습니다.
- 충돌 최소화: 기능 단위로 쪼개어 작업하면, 다른 팀원의 코드와의 충돌 가능성이 낮아집니다.

### 큰 PR의 단점

- 리뷰 부담 증가: 너무 많은 코드 변경이 한꺼번에 포함되면, 리뷰어가 놓치기 쉬운 세부사항이 많아집니다.
- 리뷰 시간 증가: 긴 PR은 리뷰하는 데 많은 시간이 소요되어, 전체 개발 프로세스에 지연을 초래할 수 있습니다.

## 좋은 PR의 구체적인 기준

#### 적절한 크기

- 라인 수: 일반적으로 한 PR은 200~400줄 정도가 이상적이라는 의견이 많습니다. 특히 200~300줄 정도의 변경 사항이면 리뷰어가 부담 없이 전체 코드를 살펴보고 구체적인 피드백을 줄 수 있습니다.

- 리뷰 시간: 리뷰에 소요되는 시간이 1시간 이내가 되는 것을 목표로 하세요. 시간이 길어지면 집중력이 떨어지므로, PR 크기를 조절하는 중요한 기준이 됩니다.

#### 한 기능 단위로 분리

- PR은 하나의 명확한 목적(예: 하나의 기능 추가, 버그 수정, 리팩토링 등)을 가져야 합니다. 이렇게 하면 변경 사항의 범위와 의도가 명확해져 리뷰어가 빠르게 이해할 수 있습니다.
- 관련 커밋은 적절하게 그룹화하고, 커밋 메시지도 “왜” 이런 변경이 필요한지 간략히 서술해야 합니다.

#### 명확한 설명과 문서화

- PR 설명: PR의 목적, 주요 변경 사항, 테스트 방법 등을 상세하게 작성합니다. 이는 리뷰어가 전체 맥락을 파악하는 데 큰 도움이 됩니다.
- 커밋 메시지: 각 커밋은 “무엇을”, “왜” 변경했는지를 명확히 기록하여, 추후 히스토리 관리와 롤백 시에도 유용하게 활용될 수 있도록 합니다.

#### 의 규칙에 따른 유연한 적용

- 모든 상황에 동일한 기준을 적용하기보다는, 프로젝트 특성이나 팀의 상황에 맞게 PR의 크기와 범위를 조절해야 합니다.
- 신규 기능 개발이나 복잡한 리팩토링의 경우, 자연스럽게 PR이 커질 수 있으므로 이 경우에는 PR 템플릿, Draft PR 기능 등을 활용해 중간 피드백을 받는 것도 좋은 방법입니다.

## 상황에 따른 유연한 접근 방법

- 신규 기능 개발: 초기 단계에서는 전체 기능이 한꺼번에 나오기 어려울 수 있습니다. 이때는 Draft PR을 활용해 기능의 틀을 먼저 공유하고, 점진적으로 리뷰를 받아가며 개발하는 방법이 효과적입니다.
- 리팩토링: 변경 범위가 넓어지면 여러 개의 작은 PR로 분리하거나, 커밋 단위로 잘 정리해서 리뷰어가 한눈에 파악할 수 있도록 해야 합니다.
- 팀의 피드백 문화: 팀 내에서 정해진 PR 규칙이나 템플릿을 활용하면, 모두가 일관된 기준으로 코드를 리뷰받을 수 있고, 이는 협업의 효율성을 크게 높여줍니다.

## 정리

좋은 PR은 단순히 “코드 변경”의 기록이 아니라, 리뷰어가 빠르고 정확하게 이해할 수 있도록 ‘의미 있는 작업 단위’로 정리된 산출물입니다.

- 적절한 크기와 명확한 목적, 그리고 체계적인 설명은 코드 리뷰의 효율성을 높여주며, 결과적으로 팀 전체의 코드 품질과 협업 문화를 개선하는 데 큰 역할을 합니다.
- 팀 내에서 이러한 기준을 공유하고 지속적으로 피드백을 주고받는 문화가 자리잡는다면, 자연스럽게 생산성과 코드 퀄리티 모두 향상될 것입니다.
