title: "실행 컨텍스트별 this의 참조 대상"
date: "241130"
tags: ["JavaScript"]
---

| 실행 컨텍스트             | this가 가리키는 대상            | 설명 및 예시                                                                                                                             |
| ------------------------- | ------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| 전역 컨텍스트             | 전역 객체 (`window` / `global`) | 최상위 코드에서 this는 브라우저에서는 `window`, Node.js 환경에서는 `global` 객체를 가리킵니다.                                           |
| 일반 함수 (non-strict)    | 전역 객체                       | 일반 함수 호출 시 (strict 모드가 아닐 때) this는 전역 객체를 참조합니다. 예: `function foo() { console.log(this); }`                     |
| 일반 함수 (strict mode)   | `undefined`                     | `'use strict';`를 사용하면, 함수 내부에서 this는 기본적으로 `undefined`가 됩니다.                                                        |
| 메소드 호출               | 해당 객체                       | 객체의 메소드로 호출할 경우, this는 그 메소드를 소유한 객체를 가리킵니다. 예: `obj.method()`에서 this는 `obj`                            |
| 생성자 함수 (Constructor) | 새로 생성된 인스턴스            | `new` 키워드를 사용해 생성자 함수를 호출하면, this는 새로 생성된 객체를 가리킵니다. 예: `new Person()`                                   |
| call / apply / bind       | 명시한 객체                     | 이들 메서드를 사용하여 함수를 호출할 경우, 첫 번째 인자로 전달한 객체를 this로 강제 설정할 수 있습니다.                                  |
| Arrow 함수                | 외부 렉시컬 환경의 this         | Arrow 함수는 자체적인 this 바인딩이 없으며, 정의 당시의 상위 스코프의 this를 그대로 사용합니다.                                          |
| 이벤트 핸들러             | 이벤트가 발생한 요소            | 브라우저 이벤트 핸들러 내부에서 this는 이벤트를 발생시킨 요소(예: 버튼, 입력 요소 등)를 가리킵니다. (단, strict mode에서는 다를 수 있음) |

#### 부연 설명

- 전역 컨텍스트와 함수 호출  
  전역에서 실행되는 코드나 일반 함수 호출의 경우, non-strict 모드에서는 `this`가 전역 객체를 참조합니다.  
  하지만 strict 모드에서는 안전하게 `undefined`를 반환하여, 실수로 전역 객체에 접근하는 것을 방지할 수 있습니다.

- 메소드와 생성자 함수  
  객체의 메소드 내부에서의 `this`는 해당 객체를 가리키며, 생성자 함수에서는 새로운 객체 인스턴스에 바인딩됩니다.  
  이 두 경우는 객체 지향 프로그래밍에서 중요한 역할을 합니다.

- call, apply, bind  
  이 메서드들은 함수 호출 시 this의 값을 명시적으로 설정할 수 있도록 도와줍니다.  
  이를 통해 함수 재사용성이 높아지고, 원하는 컨텍스트에서 함수를 실행할 수 있습니다.

- Arrow 함수의 렉시컬 this  
  Arrow 함수는 일반 함수와 달리 자신만의 this를 가지지 않고, 정의된 환경(렉시컬 환경)의 this를 그대로 사용합니다.  
  따라서, 내부 중첩 함수에서 외부 함수의 this를 그대로 사용할 때 매우 유용합니다.

- 이벤트 핸들러의 this  
  이벤트 리스너 내부의 this는 이벤트가 발생한 요소를 참조하게 됩니다.  
  이는 DOM 조작 시 유용하게 활용되지만, 상황에 따라 다른 방식으로 바인딩되기도 하므로 주의해야 합니다.
