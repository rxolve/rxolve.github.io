---
title: "git reset: 다른 사람의 브랜치에 실수로 push 했을 때"
date: "240703"
tags: ["Git"]
---

- PR 리뷰를 하다가 이슈가 생겼습니다.
- 다른 사람의 브랜치에 실수로 push 를 했습니다.
- 이때 필요한 명령어는 `git reset` 입니다.
- 모드별로 다음과 같이 사용할 수 있습니다.
  - --soft
    - 선택한 커밋까지 커밋 히스토리를 되돌리지만, 변경 사항은 스테이징 영역에 그대로 남겨둡니다.
    - HEAD는 현재 체크아웃된 커밋을 가리키며, HEAD~1은 바로 이전 커밋을 의미합니다.
      ```bash
      git reset --soft HEAD~1
      ```
    - 되돌리고자 하는 커밋은 해시 값으로도 지정할 수 있습니다.
      ```bash
      git reset --soft 1a2b3c4d
      ```
  - --mixed
    - 커밋 히스토리를 되돌리고 스테이징 영역을 초기화하지만, 작업 디렉토리의 파일은 변경되지 않습니다.
  - --hard
    - 커밋 히스토리와 스테이징 영역, 작업 디렉토리의 파일 모두를 선택한 커밋으로 되돌립니다.
- 저는 `git reset --hard HEAD~2` 을 사용하여 최근 두 개의 커밋을 되돌렸습니다.
- git reset 은 히스토리를 조작하는 강력한 명령어입니다.
  - 사용 시 주의가 필요하며, 특히 --hard 모드는 데이터 손실의 위험이 있으므로 신중하게 사용해야 합니다.
  - 저도 손을 떨면서 타이핑했습니다.
