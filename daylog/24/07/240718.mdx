## AngularJS 에 대한 회상

- 제가 웹 프론트 개발을 처음 시작한 프로젝트는 AngularJS 로 만든 것이었습니다.
- 그 당시에도 React 는 급부상중이었고, AngularJS 는 레거시 취급을 받았습니다.
- 시간이 더 흘러 AngularJS 는 지원이 종료되고, Angular 로 대체되었습니다.
- AngularJS 가 남긴 유산과 그것이 React 와 어떻게 다른지에 대해 이야기해보려고 합니다.

## AngularJS와 React 비교

- AngularJS 는 2010년에 출시되어 한때 프론트엔드 개발의 강자였지만, 현재는 역사의 뒤안길로 사라졌습니다.
- 그러나 그 영향력은 여전히 남아있습니다. React 와 비교해보면서 AngularJS 의 유산을 살펴봤습니다.

### 1. 컴포넌트 기반 아키텍처

#### AngularJS

- 처음으로 컴포넌트 기반 아키텍처의 개념을 대중화했습니다.
- 디렉티브(directive)를 통해 재사용 가능한 UI 요소를 만들 수 있었습니다.
  - 디렉티브: DOM 을 확장하는 AngularJS 의 핵심 기능. 커스텀 태그나 속성을 정의할 수 있습니다.

#### React

- 컴포넌트 기반 접근 방식을 더욱 간소화하고 유연하게 만들었습니다.
- JSX 를 통해 더 직관적인 컴포넌트 작성이 가능해졌습니다.

### 2. 데이터 바인딩

#### AngularJS

- 양방향 데이터 바인딩을 도입했습니다. 모델의 변경사항이 뷰에 자동으로 반영되고, 그 반대도 가능했죠.
- ng-model 디렉티브를 사용해 쉽게 구현할 수 있었습니다.

#### React

- 단방향 데이터 흐름을 채택했습니다. 이는 애플리케이션의 상태 관리를 더 예측 가능하게 만듭니다.
- setState()나 훅을 사용해 상태를 명시적으로 업데이트해야 합니다.

### 3. 의존성 주입

#### AngularJS

- 의존성 주입 시스템을 프론트엔드 개발에 도입했습니다.
- 이를 통해 모듈 간의 결합도를 낮추고 테스트 용이성을 높였습니다.
- NestJS 와 같은 백엔드 프레임워크에도 영향을 주었습니다.

#### React

- 의존성 주입 대신 props 를 통한 데이터 전달 방식을 사용합니다.
- Context API 나 외부 상태 관리 라이브러리(예: Redux)를 통해 비슷한 효과를 얻을 수 있습니다.

### 4. 템플릿 문법

#### AngularJS

- HTML 을 확장한 템플릿 문법을 사용했습니다. (예: ng-if, ng-repeat)
- 이는 기존 웹 개발자들이 쉽게 적응할 수 있는 장점이 있었습니다.

#### React

- JSX 를 도입하여 JavaScript 내에서 직접 UI를 표현할 수 있게 했습니다.
- 이는 더 강력한 표현력을 제공하지만, 학습 곡선이 높아졌습니다.

### 5. 성능

#### AngularJS

- 양방향 데이터 바인딩과 다이제스트 사이클로 인해 큰 애플리케이션에서 성능 이슈가 있었습니다.
  - 다이제스트 사이클: AngularJS 가 모델 데이터 변경을 감지하고 해당 변경 사항을 뷰에 반영하는 과정.

#### React:

- 가상 DOM을 도입하여 효율적인 UI 업데이트를 가능하게 했습니다.
- 이는 대규모 애플리케이션에서 더 나은 성능을 제공합니다.

## Angular 로의 전환

- AngularJS 는 출시 이후 빠르게 인기를 얻었지만, 시간이 지나면서 몇 가지 한계점이 드러났습니다.
- 이에 Google 팀은 완전히 새로운 프레임워크인 Angular(Angular 2+)를 개발하기로 결정했습니다.

### Angular의 주요 변화

- TypeScript 사용: 정적 타입 지원으로 대규모 애플리케이션 개발과 협업에 더 적합해졌습니다.
- 성능 개선: 변화 감지 메커니즘의 개선으로 성능이 크게 향상되었습니다.
- 모바일 지원: 모바일 개발을 위한 최적화가 이루어졌습니다.
- 컴포넌트 기반 아키텍처 강화: AngularJS 의 컨트롤러와 스코프 개념이 제거되고, 컴포넌트 중심으로 재설계되었습니다.

### Angular 가 AngularJS 만큼 인기를 얻지 못한 이유

- 마이그레이션의 어려움: AngularJS 에서 Angular 로의 전환이 사실상 새로운 프레임워크를 배우는 것과 다름없이 어려웠습니다.
- React의 부상: 같은 시기에 React 가 단순성과 유연성을 무기로 빠르게 성장하고 있었습니다.
- 생태계 변화: AngularJS 시대에 구축된 많은 라이브러리와 도구들이 Angular 와 호환되지 않았습니다.
- 버전 관리의 혼란: Angular2 이후 빠른 메이저 버전 업데이트로 개발자들이 따라가기 힘들었던 것도 한 몫했습니다.

## 결론

- AngularJS 는 많은 혁신적인 아이디어를 프론트엔드 개발에 도입했습니다. 컴포넌트 기반 아키텍처, 데이터 바인딩, 의존성 주입 등의 개념은 현대 프론트엔드 프레임워크의 기반이 되었으니까요. React 는 이러한 개념들을 바탕으로, 더 유연하고 성능이 좋은 라이브러리를 만들어냈고, 현재는 프론트엔드 라이브러리에 절대강자로 자리잡았습니다.
- Angular 로의 전환 과정에서 겪은 어려움은 기술 선택의 중요성과 개발자 커뮤니티의 역할을 잘 보여줍니다. 새로운 기술을 도입할 때는 학습 곡선, 마이그레이션 용이성, 생태계 지원 등을 종합적으로 고려해야 한다는 것을 알 수 있었습니다.
- AngularJS 가 남긴 유산을 아는 것은 현대 프론트엔드 개발의 진화 과정을 이해하는 데 큰 도움이 되었습니다. React 를 비롯한 현대 프레임워크들이 왜 이런 방식으로 설계되었는지, 어떤 문제들을 해결하려 했는지 가늠할 수 있었습니다.
