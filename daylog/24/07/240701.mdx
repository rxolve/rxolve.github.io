>
> - 객체지향 패러다임의 탄생부터 원칙들과 방법론 그리고 자바스크립트에서 객체지향을 흉내내기 위한 눈물겨운 노력을 다룬 글입니다.

### 프로토타입 언어

- 자바스크립트를 OOP를 잘쓰기 위해선 먼저 **프로토타입** 언어를 이해해야 합니다.
- 클래스 기반으로 객체지향이 발전할 때, 한쪽에선 프로토타입 기반으로 객체지향을 구현한 언어들이 있었습니다.

#### Self (1980년대 후반)

- 프로토타입 기반 객체지향 언어의 시초입니다.
- 클래스 대신 프로토타입을 사용하여 객체를 정의하고, 객체를 복제하여 새로운 객체를 생성하는 방식을 채택했습니다.

#### Lua (1993년)

- C/C++ 프로그램 내부에 포함시키기 매우 쉬운 깔끔한 문법의 가벼운 스크립트 언어를 목표로 개발되었습니다.

#### JavaScript (1995년)

- 가장 유명한 프로토타입 언어입니다.
- 태생인 웹 브라우저를 넘어 서버 사이드, 모바일 앱, 데스크탑 등 다양한 환경에서 사용되고 있습니다.

#### Dart (2011년)

- 플러터를 위해 개발되었으며, 자바스크립트의 단점을 보완하고자 만들어졌습니다.

#### TypeScript (2012년)

- 타입을 지원하는 자바스크립트의 슈퍼셋입니다.

### 프로토타입 체인

- 프로토타입 언어의 가장 큰 특징은 클래스에서 객체를 생성하는 것이 아니라, 객체에서 새로운 객체를 생성한다는 것입니다.
- 이 객체들은 프로토타입 체인을 통해 연결되어 있습니다.
- 객체가 특정 속성이나 메서드를 찾지 못하면, 이 체인을 따라 상위 객체에서 이를 찾습니다.

```javascript
const person = {
  sayHello() {
    console.log(`Hello, my name is ${this.name}`);
  },
};

const alice = Object.create(person);
alice.name = "Alice";

const employee = Object.create(alice);
employee.jobTitle = "Developer";
employee.sayJobTitle = function () {
  console.log(`My job title is ${this.jobTitle}`);
};

employee.sayHello(); // "Hello, my name is Alice"
employee.sayJobTitle(); // "My job title is Developer"
```

- 이 체인을 통해 객체지향의 상속을 포함한 다형성, 캡슐화, 추상화 등의 개념을 구현했습니다.

### 자바스크립트에서 OOP의 어려움

여기서 자바와 같은 클래스 기반 객체지향 언어를 쓰던 사람들에게 혼동을 가져올 수 있습니다.

- ES6에서 class 키워드가 도입되었지만, 이는 프로토타입 기반 상속을 감싸는 Syntax Sugar 에 불과합니다.
  - 프로토타입 기반의 본질을 바꾸지 않기 때문에, 상속이나 메서드 오버라이딩 같은 구현에서 예상과 다른 동작을 할 수 있습니다.
- 객체에 속성이나 메서드를 추가하거나 변경하기 쉽습니다. 클래스 기반 언어에 비해 규칙이 느슨합니다.
- 자바스크립트는 다중 상속을 직접적으로 지원하지 않습니다.

  - 믹스인 패턴 등으로 해결해야 하는데, 추가적인 코드 작성이 필요하고 구조의 복잡성을 증가시킵니다.

  ```javascript
  // 믹스인 함수 정의
  function mixin(target, ...sources) {
    Object.assign(target.prototype, ...sources);
  }

  ...

  // 믹스인 적용
  mixin(Animal, canSwim, canFly);

  ```

- 무엇보다 자바스크립트는 함수형 프로그래밍 패러다임을 강력하게 지원합니다.

  - 이로 인해 개발자들은 함수형으로 구현을 더 선호하게 되고, 이는 객체지향관점에서 다소 멀어지게 되기 쉽습니다.

자바스크립트의 가장 큰 매력은 다양한 패러다임을 자유롭게 사용할 수 있는 유연함이라고 생각합니다.

테오님 포스팅의 마지막 전 문단처럼

- 데이터와 메소드를 기반으로 작은 프로그램을 만들어 코드의 재사용성을 높이고
- 간결한 문법의 장점은 취하고
- 복잡한 구조는 지양하고
- 함수형 프로그래밍의 장점과 결합하는 것이

자바스크립트에서 객체지향을 잘한다고 볼 수 있을 것 같습니다.
