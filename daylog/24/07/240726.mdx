shadcn은 기존의 UI 라이브러리들과 어떻게 다른 접근 방식으로 개발자들의 관심을 끌고 있는지 궁금했습니다.

## shadcn이란?

shadcn은 React 기반의 UI 컴포넌트 모음입니다. 하지만 일반적인 npm 패키지 형태의 라이브러리가 아닌, "복사-붙여넣기" 방식의 컴포넌트 제공 방식을 채택하고 있습니다.

## 주요 특징

### 복사-붙여넣기 모델

shadcn의 가장 큰 특징은 컴포넌트 코드를 직접 프로젝트에 복사하여 사용한다는 점입니다. 이를 통해 커스터마이징 자유도를 높이고, 프로젝트에 대한 완전한 제어권을 가질 수 있습니다.

### Tailwind CSS 기반 스타일링

shadcn은 Tailwind CSS를 기본 스타일링 도구로 사용합니다. 빠른 스타일링과 프로토타이핑이 가능하고, Tailwind의 유틸리티 클래스를 활용한 일관된 디자인 시스템을 구축할 수 있습니다.

### 접근성 중심 설계

모든 컴포넌트가 높은 수준의 웹 접근성 표준을 준수하도록 설계되었습니다.

### 헤드리스 접근 방식

헤드리스 UI 패턴을 채택하여 개발자가 원하는 대로 스타일링이 가능합니다.

- 헤드리스 UI 패턴: 기능과 스타일을 분리하여 컴포넌트의 재사용성과 유연성을 높이는 방식

### 구성 요소 중심

역할이 아닌 구성 요소를 중심으로 설계되어 있습니다. 이에 따라 추상화 수준을 높여 더 유연한 컴포넌트 설계가 가능합니다. 재사용 가능한 기본 구성 요소를 제공하기에, 레고 블록처럼 조합하여 복잡한 UI 또한 쉽게 구현할 수 있습니다.

### 학습 리소스

잘 짜여진 컴포넌트의 내부 구현을 직접 볼 수 있어 학습에 도움을 줍니다. 컴포넌트의 수정을 통해 최신 웹 개발 기술과 패턴을 학습할 수 있습니다.

## shadcn/ui과 Pico CSS 비교

이 블로그는 Pico CSS를 사용하고 있습니다. Pico와 shadcn의 차이를 비교해보겠습니다.

### 기본 철학

- **Pico**: '클래스리스(classless)' CSS 프레임워크로, 최소한의 설정으로 깔끔한 디자인을 제공합니다.
- **shadcn**: 커스터마이즈 가능한 컴포넌트 라이브러리로, 개발자에게 더 많은 제어권을 제공합니다.

### 사용 방식

- **Pico**: CSS 파일을 링크하거나 설치하여 즉시 사용 가능합니다.
- **shadcn**: 필요한 컴포넌트를 프로젝트에 복사하여 사용합니다.

### 커스터마이징

- **Pico**: CSS 변수를 통해 기본적인 커스터마이징이 가능합니다.
- **shadcn**: 컴포넌트 코드에 직접 접근하여 높은 자유도의 커스터마이징이 가능합니다.

### 타겟

- **Pico**: 빠른 프로토타이핑이나 간단한 프로젝트에 적합합니다.
- **shadcn**: 복잡한 UI와 고도의 커스터마이징이 필요한 대규모 프로젝트에 적합합니다.

### 학습 곡선

- **Pico**: 매우 낮은 학습 곡선으로 빠르게 적용 가능합니다.
- **shadcn**: React와 Tailwind CSS에 대한 이해가 필요하며, 상대적으로 높은 학습 곡선을 가집니다.

## shadcn/ui과 Pico CSS 함께 사용하기

기본 컴포넌트에 Pico를 사용하고, 커스터마이징 하고픈 UI 컴포넌트에 shadcn을 사용하는 전략을 세워봤습니다. 하지만 충돌하는 스타일을 해결할 수 없었습니다.

```css
// globals.css
@import url("https://cdn.jsdelivr.net/npm/@picocss/pico@2/css/pico.min.css");

@tailwind utilities;

// shadcn/ui를 사용하려면 여기에 아래 디렉티브를 임포트해야 합니다.
// @tailwind base;
// @tailwind components;
// 이렇게 하면 Pico가 깨집니다.
```

결국 shadcn을 반절만 활용할 수 있을 것 같습니다.

## 마치며

shadcn은 현대적인 웹 개발의 트렌드를 잘 반영하면서도, 독특한 접근 방식으로 개발자들에게 새로운 가능성을 제시하고 있습니다.

Pico와 같은 간편한 프레임워크와 비교했을 때, shadcn은 더 높은 수준의 커스터마이징과 복잡한 UI 구현에 강점을 보입니다.

특히 프로젝트에 대한 완전한 제어권을 원하는 개발자들의 특성으로 shadcn은 매력적인 도구가 되었단걸 알 수 있었습니다.

아직 Pico와 Tailwind의 조합을 사용하고 있지만, 프로젝트의 복잡도가 증가함에 따라 shadcn을 부분적으로 도입하는 것을 고려해보고 있습니다.
