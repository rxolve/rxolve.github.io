`.cursorrules` 파일은 Cursor AI 에디터에 프로젝트별 규칙을 정의하여, AI가 여러분의 코드 스타일과 아키텍처를 정확히 이해하고 일관성 있는 코드를 생성하도록 돕습니다.  
전역 규칙(Global Rules)과 프로젝트 규칙(Project Rules)으로 나누어 설정할 수 있으며, 간단한 문법으로 팀의 코딩 컨벤션, 파일 구조, 테스트 커버리지 등 다양한 요구사항을 반영할 수 있습니다.  
이를 통해 코드 리뷰 시간을 단축하고, 신규 팀원의 온보딩을 가속화하며, 반복 작업을 자동화할 수 있습니다.

## `.cursorrules` 의 개념

- 전역 규칙(Global Rules): Cursor 설정(Settings)에서 작성하며, 모든 프로젝트에 적용됩니다.
- 프로젝트 규칙(Project Rules): 프로젝트 루트의 `.cursorrules` 파일에 작성하며, 해당 프로젝트에만 적용됩니다.

## 설정 파일 만들기

1. 프로젝트 루트에 `.cursorrules` 파일 생성
2. 규칙을 Markdown 형태로 작성
3. Cursor 설정에서 “Include `.cursorrules` file” 옵션 활성화

```markdown
# .cursorrules 예시

## 코드 스타일

- 항상 TypeScript strict 모드 사용
- 인터페이스 선언을 명시적으로 작성
- `any` 타입 사용 지양

## 컴포넌트 개발 (React)

- 함수형 컴포넌트만 허용
- Props에 대해 항상 타입 정의
- CSS Modules 사용

## 테스트

- 유닛 테스트 커버리지 80% 이상
- React Testing Library 사용
```

## 주요 규칙 유형 및 예시

#### 코드 컨벤션

- 네이밍: camelCase 변수, PascalCase 컴포넌트
- 파일 구조: `components/`, `hooks/`, `utils/` 등 디렉터리 구분
- 스타일: 줄 길이 80자, 세미콜론 사용 여부, 탭/스페이스

#### 아키텍처 가이드

- 모듈별 책임 분리: “한 파일에 하나의 책임”
- 계층 구조: `domain/`, `application/`, `infrastructure/` 패턴

#### 자동화 및 검사

- 테스트 커버리지 문구
- CI 파이프라인용 후크: 커밋 전 린트·유닛테스트 실행

## 예시 시나리오

#### NestJS API 프로젝트

```markdown
# NestJS 규칙

- Controller는 최대 10줄 이내
- Service 레이어에서 직접 DB 접근 금지 (Repository 통해서만)
- DTO는 항상 클래스-validator 적용
```

#### Next.js 정적 블로그

```markdown
# Next.js 블로그

- 페이지 파일: `src/pages//*.tsx`
- `getStaticProps` 반드시 사용
- `Image` 컴포넌트로 모든 이미지 렌더링
```

## 팀 협업과 유지보수

- 버전 관리: `.cursorrules` 파일도 Git에 포함
- 문서화: README.md에 규칙 요약
- 온보딩: 신규 팀원에게 규칙 파일 읽도록 안내

## 활용 팁

- 조건부 규칙: 파일 패턴별로 세분화 (`@file` 지시자 사용)
- 외부 참조: `.cursor/rules/*.mdc` 형태로 관리하며, 모듈화
- 대화형 피드백: 규칙 위반 시 커서가 자동 제안하도록 설정

## 결론

`.cursorrules`는 단순히 AI 도구의 설정 파일이 아니라, 팀의 개발 철학을 그대로 반영하는 “코딩 헌장”입니다.  
잘 정의된 규칙 하나가 코드 품질을 10배 높이고, 리뷰 시간을 대폭 단축하며, 팀 전체의 생산성을 극대화할 수 있습니다.
