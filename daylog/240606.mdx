> [How to deploy a Nextjs app to GitHub pages?](https://medium.com/frontendweb/how-to-deploy-a-nextjs-app-to-github-pages-1de4f6ed762e)

0. 어제에 이어 이곳을 [https://rxolve-github-io.vercel.app](https://rxolve-github-io.vercel.app) 에서 [https://rxolve.github.io](https://rxolve.github.io) 로 변경하는 작업을 진행했다.
   - 먼저 어제 작업사항을 되돌린다.
1. 리포지토리의 Stting - Pages - Source 를 Github Actions로 변경한다.
2. configure 버튼을 눌러 기본 workflow 파일을 생성한다.
3. 푸시를 해 빌드를 돌리니 Setup Pages 단계에서 에러가 발생했다. MDX 를 빌드하는 과정에서 제대로 작동하지 않는 것 같다.
   > [Mdx does not build properly with GitHub Pages (Actions) #58637](https://github.com/vercel/next.js/issues/58637)
   >
   > workflow 에서 삭제해준다.
4. 어제와 동일하게 next.config.js 에 아래와 같은 설정을 추가한다.

   ```javascript
     output: 'export',
     images: {
       unoptimized: true,
     },

   ```

   - package.json 에 "type": "module" 을 명시해서 next.config.mjs 에서 next.config.js 로 돌아왔다.
   - 정적 파일을 불러오는 경로를 못 찾는다면(어제의 나처럼) production 환경에서 assetPrefix 를 추가해주면 된다고 한다.

     ```javascript
     assetPrefix:
        process.env.NODE_ENV === 'production' ? '/[repository-name]' : '',
     ```

5. main 브랜치에 푸시하면 빌드를 시작한다. 배포가 완료되니 제대로 나오는 홈페이지를 확인할 수 있었다.
   - 피코 선생님이 돌아왔다. 🎉
   - [https://rxolve-github-io.vercel.app](https://rxolve-github-io.vercel.app) 은 자동으로 비활성화가 되었다.
   - 정리하고 나니 간단한데 이 과정에서 푸시한 커밋이 14개나 된다. 😅 GPT-4o 선생님을 너무 믿었다.
