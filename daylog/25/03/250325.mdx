#### 동시성 (Concurrency)

- 정의:  
  여러 작업이 논리적으로 동시에 실행되는 것처럼 보이도록 설계하는 개념입니다.  
  실제로는 한 작업의 일부가 실행되고, 빠른 컨텍스트 스위칭을 통해 다른 작업의 일부가 실행되어 "동시에" 수행되는 것처럼 보입니다.
- 특징:
  - 싱글 코어에서도 가능: 단일 CPU 코어에서도 여러 작업이 시간 분할(time slicing) 방식으로 실행될 수 있습니다.
  - 구조적 접근: 시스템 설계 시 여러 작업을 관리하는 방식에 초점을 맞추며, 프로그램의 복잡성을 다루기 위한 패턴(예: 이벤트 루프, 비동기 프로그래밍)을 포함합니다.
  - 자원 공유 이슈: 동시 실행되는 작업들이 동일한 자원에 접근할 경우, race condition이나 데드락 등 동기화 문제가 발생할 수 있습니다.

> 예를 들어, Node.js의 비동기 I/O는 단일 스레드 내에서 여러 요청을 동시에 처리하는 동시성 모델을 사용합니다.

#### 병렬성 (Parallelism)

- 정의:  
  여러 작업이 물리적으로 동시에 실행되는 것을 의미합니다.  
  즉, 멀티코어 또는 다중 CPU를 활용하여 여러 작업이 동시에 진행됩니다.
- 특징:
  - 실제 동시 실행: 여러 코어가 존재할 경우, 각 코어가 다른 작업을 독립적으로 처리하여 진정한 병렬 처리를 이룹니다.
  - 성능 향상: 작업을 병렬 처리하면 CPU의 효율을 극대화하고 전체 처리 시간을 줄일 수 있습니다.
  - 제한: 모든 작업이 병렬 처리될 수 있는 것은 아니며, 작업 간 의존성이나 I/O 바운드 작업의 경우 동시성이 더 효과적일 수 있습니다.

> 예를 들어, 데이터 병렬 처리나 멀티스레딩을 통한 계산 집약적 작업은 멀티코어 시스템에서 병렬적으로 실행되어 성능을 향상시킵니다.

#### 동시성과 병렬성의 관계 및 차이

- 동시성은 설계 철학:  
  여러 작업을 효율적으로 관리하고 스케줄링하는 논리적인 개념입니다.  
  단일 프로세서 환경에서도 동시성을 구현할 수 있습니다.
- 병렬성은 실행 방식:  
  실제로 여러 작업이 동시에 실행되는 물리적인 개념입니다.  
  이를 위해서는 멀티코어나 여러 CPU가 필요합니다.
- 포함 관계:  
  병렬 처리는 동시성의 한 형태라고 볼 수 있습니다.  
  즉, 병렬성은 동시성의 하위 개념 중 하나로, 동시성을 활용하여 실제로 작업을 병렬로 수행하는 경우라고 할 수 있습니다.

#### 정리

- 동시성(Concurrency):  
  여러 작업을 논리적으로 관리하여 동시에 진행되는 것처럼 보이게 하는 설계 방식.  
  단일 코어에서도 시간 분할을 통해 구현 가능하며, 주로 I/O 바운드 작업에서 효과적입니다.
- 병렬성(Parallelism):  
  여러 작업을 물리적으로 동시에 실행하는 방식으로, 멀티코어나 다중 CPU 환경에서 주로 사용됩니다.  
  계산 집약적 작업에 유리하며 실제 성능 향상을 도모합니다.
