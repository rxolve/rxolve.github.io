## 쿠버네티스 리소스?

쿠버네티스 리소스는 클러스터 내에서 애플리케이션과 인프라를 구성하고 관리하기 위한 “객체(오브젝트)”를 정의하는 틀입니다.  
이는 객체지향 프로그래밍에서 클래스와 인스턴스의 관계와 유사합니다.  
즉, 리소스는 객체를 생성할 때 사용하는 설계도(클래스)이고, 실제 생성된 객체들이 바로 오브젝트입니다.

## 주요 리소스 및 역할

#### Pod

- 설명: 쿠버네티스에서 가장 작은 실행 단위로, 하나 이상의 컨테이너를 포함합니다.
- 특징: 동일한 네트워크 네임스페이스와 스토리지를 공유하며, 주로 애플리케이션의 단일 인스턴스로 동작합니다.

#### ReplicaSet

- 설명: 지정된 수의 Pod 복제본을 유지하는 역할을 합니다.
- 특징: Pod가 기대한 개수보다 줄어들면 자동으로 새로운 Pod를 생성하고, 반대로 과도한 경우 삭제합니다.

#### Deployment

- 설명: ReplicaSet을 관리하는 상위 리소스로, 애플리케이션 배포와 업데이트(롤링 업데이트, 롤백 등)를 쉽게 수행할 수 있도록 도와줍니다.
- 특징: 선언형 방식으로 원하는 상태를 정의하면, 클러스터 내에서 실제 상태가 자동으로 조정됩니다.

#### StatefulSet

- 설명: 상태 정보를 유지해야 하는 애플리케이션에 사용됩니다.
- 특징: 각 Pod에 고유한 네트워크 식별자와 스토리지 볼륨을 제공하여, 데이터베이스나 메시지 큐 같은 상태 저장 애플리케이션에 적합합니다.

#### DaemonSet

- 설명: 클러스터의 모든(또는 특정) 노드에서 단 하나의 Pod가 실행되도록 보장합니다.
- 특징: 로그 수집, 모니터링 에이전트 등 시스템 수준의 서비스를 배포할 때 유용합니다.

#### Job & CronJob

- Job
  - 설명: 일회성 배치 작업을 실행하여 완료되면 종료되는 리소스입니다.
- CronJob
  - 설명: 일정 주기로 반복 실행해야 하는 작업을 관리합니다.

#### Service & Ingress

- Service
  - 설명: Pod 집합에 대한 안정적인 네트워크 접근점을 제공합니다.
  - 특징: 로드 밸런싱 및 서비스 디스커버리 기능을 수행합니다.
- Ingress
  - 설명: 외부 트래픽을 적절한 서비스로 라우팅하는 역할을 합니다.
  - 특징: 도메인 기반, 경로 기반 라우팅 등 정교한 트래픽 제어가 가능합니다.

#### ConfigMap & Secret

- ConfigMap
  - 설명: 애플리케이션의 일반 설정 정보를 관리합니다.
- Secret
  - 설명: 암호, API 키 등 민감한 정보를 안전하게 저장합니다.
- 특징: 두 리소스 모두 애플리케이션에 환경 변수나 볼륨 등으로 주입할 수 있습니다.

#### PersistentVolume (PV) & PersistentVolumeClaim (PVC)

- PV
  - 설명: 클러스터 관리자가 제공하는 실제 스토리지 자원을 추상화한 객체입니다.
- PVC
  - 설명: 사용자(또는 애플리케이션)가 스토리지 사용 요청을 할 때 사용하는 객체입니다.
- 특징: 이들 리소스를 통해 컨테이너의 데이터가 클러스터 재시작 이후에도 유지됩니다.

## 관리 및 배포 방식

쿠버네티스 리소스는 주로 YAML 파일로 선언하여 관리합니다.  
예를 들어, Deployment를 정의할 때 아래와 같이 YAML 파일을 작성한 후,

```bash
kubectl apply -f deployment.yaml
```

명령어로 클러스터에 적용할 수 있습니다.

또한, 리소스의 상태 확인, 업데이트, 삭제 등은 kubectl 명령어를 통해 수행할 수 있습니다.

- `kubectl get pods` : 실행 중인 Pod 목록 확인
- `kubectl describe deployment <deployment-name>` : 특정 Deployment 상세 정보 확인
- `kubectl delete svc <service-name>` : 서비스 삭제
