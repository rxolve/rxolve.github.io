#### 타입 안전성

문자열 리터럴 타입을 사용하면 변수에 허용되는 값이 미리 정해진 집합으로 한정됩니다.  
예를 들어, 상태를 나타내는 타입을 아래와 같이 정의할 수 있습니다.

```typescript
type Status = "pending" | "approved" | "rejected";
let currentStatus: Status = "pending"; // 올바른 사용
currentStatus = "processing"; // 컴파일 오류 발생!
```

이처럼, 잘못된 값이 입력되면 컴파일 타임에서 즉시 오류가 발생합니다.  
이는 런타임 에러를 사전에 방지할 수 있어 안정적인 코드를 작성하는 데 큰 도움이 됩니다.

#### 자가 문서화

문자열 리터럴 타입은 해당 변수나 함수 인자가 어떤 값들을 허용하는지 명확하게 보여줍니다.  
예를 들어, 이벤트 핸들러를 구현할 때 아래와 같이 작성할 수 있습니다.

```typescript
type MouseEventName = "click" | "dblclick" | "mousedown" | "mouseup";

function registerEventHandler(event: MouseEventName, handler: () => void) {
  // 이벤트 등록 로직
}

registerEventHandler("click", () => console.log("클릭!")); // OK
registerEventHandler("mouseover", () => console.log("마우스오버!")); // 컴파일 오류!
```

코드만 봐도 “여기서는 click, dblclick, mousedown, mouseup만 가능하다”라는 의도가 전달되므로, 동료 개발자나 미래의 내가 코드를 이해하는 데 도움이 됩니다.

#### IDE 자동 완성과 생산성 향상

문자열 리터럴 타입은 IDE와 에디터가 제공하는 자동 완성 기능과 긴밀하게 연동됩니다.  
예를 들어, 위의 `MouseEventName` 타입을 사용하면 코드를 작성할 때 IDE가 가능한 값들을 미리 제시해주므로 오타를 방지하고 빠르게 코드를 작성할 수 있습니다.

이는 개발자가 반복적으로 값을 타이핑하는 부담을 줄여주며, 코드 작성 속도를 크게 향상시킵니다.

#### 리팩토링과 유지보수의 용이성

허용되는 값의 집합이 한 곳에 모여있기 때문에, 만약 값이 추가되거나 변경되어야 하는 경우 해당 타입 선언만 수정하면 됩니다.  
컴파일러는 변경된 타입에 맞지 않는 부분을 자동으로 알려주므로, 리팩토링 시 실수를 줄일 수 있습니다.

또한, 코드 전반에 걸쳐 동일한 타입을 사용하므로 코드의 일관성이 유지되고, 장기적인 유지보수가 용이해집니다.

#### 런타임 오버헤드 없이 얻는 강력한 타입 안정성

문자열 리터럴 타입은 오직 컴파일 타임에만 사용되는 타입 검사 기능입니다.  
실행 시에는 일반 문자열로 처리되므로, 별도의 런타임 코드가 추가되지 않습니다.  
즉, 안전성을 높이면서도 JavaScript 번들 크기를 불필요하게 늘리지 않는다는 장점이 있습니다.

#### 정리

문자열 리터럴 타입은 다음과 같은 여러 장점을 제공합니다.

- 타입 안전성: 미리 정의된 값만 허용되어, 잘못된 값이 들어가는 것을 방지합니다.
- 자가 문서화: 허용 가능한 값들이 명확히 드러나, 코드의 의도를 쉽게 파악할 수 있습니다.
- IDE 지원: 자동 완성 기능을 통해 개발 생산성을 크게 높여줍니다.
- 리팩토링 용이성: 변경사항이 한 곳에 모여 있어 유지보수가 편리합니다.
- 런타임 오버헤드 제로: 컴파일 타임에만 검사되므로 실행 시 성능 저하 없이 안정성을 확보할 수 있습니다.
