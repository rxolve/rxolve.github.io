## 보편자

철학에서 보편자는 여러 개별 사물이나 현상에 내재하는 “공통된 본질”을 의미합니다. 플라톤의 이데아처럼, 보편자는 각기 다른 구체체가 닮아가는 근본적인 성질입니다. 프로그래밍 세계에서 이러한 개념은 추상 클래스나 인터페이스를 통해 구현되지만, 변수명에서도 중요한 역할을 합니다.

변수명은 단순한 기호 이상의 의미를 지닙니다. 올바른 변수명은 그 자체로 보편적인 개념-즉, 코드가 다루려는 “의미”-을 드러내며, 이를 통해 코드를 읽는 이에게 공통된 이해를 제공합니다.

## 변수명에 담긴 보편적 사고

#### 변수명 선택

변수명을 결정할 때 흔히 겪는 고민은 “내가 담으려는 개념은 얼마나 추상적인가?”입니다.

- 과도하게 구체적인 변수명:  
  변수명이 지나치게 세부적인 내용을 담으면, 재사용성이나 범용성이 떨어질 수 있습니다. 예를 들어, `averageOfStudentScoresForMath`라는 이름은 해당 맥락에서는 명확할 수 있지만, 다른 용도에서는 지나치게 제한적입니다.
- 너무 추상적인 변수명:  
  반대로, `data`, `info`와 같이 너무 일반적인 이름은 그 안에 담긴 의미를 전달하지 못해 코드를 이해하기 어렵게 만듭니다.

핵심은 보편자적 관점을 유지하며, 코드의 맥락에 따라 적절한 “추상화 수준”을 선택하는 것입니다. 이름 하나에도 “이 변수는 어떤 역할을 하며, 어떤 보편적 개념을 대표하는가?” 라는 질문이 내포되어야 합니다.

#### 추상화와 구체체의 균형

보편자 개념은 추상화의 핵심이지만, 구체적인 상황에 맞게 조정되어야 합니다. 변수명을 결정할 때도 마찬가지입니다.

- 보편적 이름 (예: `result`, `counter`, `buffer`):  
  이러한 이름은 여러 상황에서 재사용될 수 있는 추상적 개념을 나타냅니다.
- 구체적 이름 (예: `userAge`, `sessionToken`):  
  해당 변수의 사용처가 명확해지고, 코드의 의도를 분명히 전달합니다.

두 접근법은 상호 보완적입니다. 보편적 개념을 기반으로 한 네이밍 전략은 코드를 모듈화하고, 추상적인 설계 원칙을 전달하는 동시에, 필요에 따라 구체적인 상황을 구현할 수 있게 합니다.

## 간단한 예제

보편적인 데이터를 처리하는 함수를 설계할 때, 변수명이 어떻게 추상성과 구체성을 동시에 반영할 수 있는지 보여줍니다.

```typescript
// 보편적 개념을 담은 추상적 변수명
let items: any[] = fetchData(); // 여기서 items는 다양한 형태의 데이터를 포함할 수 있는 보편적 컨테이너입니다.

// 데이터를 처리하는 보편적 함수
function processItems(data: any[]): any[] {
  const processed: any[] = [];
  for (const item of data) {
    // 구체적 로직은 item의 형태에 따라 다르지만, 변수명은 그 본질을 추상화하여 표현합니다.
    processed.push(transform(item));
  }
  return processed;
}

// 구체적 상황을 반영한 변수명 예제
const userProfiles: UserProfile[] = filterUserProfiles(items);
const sessionTokens: string[] = extractSessionTokens(items);
```

위 코드에서 `items`라는 변수명은 보편적인 변수명으로, 구체적인 데이터 유형이나 내용을 직접적으로 나타내지 않습니다. 대신, 함수 `processItems`와 같이 코드 전체의 맥락에 따라 그 용도를 파악할 수 있도록 유도합니다. 반면, `userProfiles`나 `sessionTokens`처럼 상황에 맞게 구체적인 이름을 사용하면, 코드가 더 명확해지고 유지보수하기 쉬워집니다.

## 정리하며

좋은 변수명은 코드 내부에 존재하는 추상적 개념들을 구체적인 요구사항과 균형 있게 연결시키며, 개발자가 담고자 하는 “의미”를 명확하게 드러내는 역할을 합니다.
