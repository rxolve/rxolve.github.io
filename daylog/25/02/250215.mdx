정지 문제는 컴퓨터 과학에서 가장 유명한 결정 불가능 문제 중 하나입니다.

#### 정지 문제

정지 문제란 "임의의 프로그램과 입력이 주어졌을 때, 해당 프로그램이 언젠가 종료될지(정지할지) 아니면 무한 루프에 빠질지를 판별하는 알고리즘이 존재하는가?"라는 질문입니다.

앨런 튜링이 1936년에 증명한 결과에 따르면, 이러한 일반적인 판별 알고리즘은 존재할 수 없다는 것이 밝혀졌습니다.

즉, 정지 문제는 결정 불가능 합니다.

#### 왜 정지 문제는 풀 수 없을까? (튜링의 대각선 논법)

튜링은 모순을 유도하는 방식으로 정지 문제가 결정 불가능함을 증명했습니다.

1. 정지 여부를 판별하는 가정된 알고리즘 H가 있다고 해봅시다.

   - `H(P, I) → True`이면 프로그램 P가 입력 I에 대해 정지한다.
   - `H(P, I) → False`이면 프로그램 P가 입력 I에 대해 무한 루프에 빠진다.

2. 이제, 특별한 프로그램 D를 만들자.

   - D는 `H(P, P)`의 결과를 보고 동작하도록 설계됩니다.
   - 만약 `H(P, P) == True`라면, D는 무한 루프에 빠집니다.
   - 만약 `H(P, P) == False`라면, D는 즉시 정지합니다.

3. D를 자기 자신에게 입력하면 (`D(D)`) 어떤 일이 일어날까요?

   - `H(D, D) == True` → D는 정지해야 하지만, 설계상 무한 루프에 빠짐 → 모순!
   - `H(D, D) == False` → D는 멈추지 않아야 하지만, 즉시 정지함 → 모순!

4. 따라서, 이러한 H는 존재할 수 없다. 즉, 정지 문제를 해결하는 알고리즘은 존재하지 않는다.

#### 정지 문제의 중요성

정지 문제는 단순한 이론 문제가 아니라, 소프트웨어 검증과 컴퓨터 과학 전반에 걸쳐 중요한 영향을 미칩니다.

- 컴파일러 & 정적 분석: 코드가 무한 루프에 빠질지 100% 보장하는 분석 도구는 존재할 수 없음.
- 보안: 악성 코드가 특정 조건에서 실행될지 예측하는 문제와도 관련됨.
- 튜링 완전성: 대부분의 프로그래밍 언어는 튜링 완전성을 가지므로, 정지 문제를 해결할 수 없음.
