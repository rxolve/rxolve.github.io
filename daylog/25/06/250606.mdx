## 정리

1. 7가지 기본 원칙
   AI에게 원하는 결과를 얻기 위해 다음 일곱 가지 원칙을 기억해야 합니다.

   - 풍부한 컨텍스트 제공: 관련 코드·환경 정보를 충분히 담습니다.
   - 목표와 질문의 구체화: “이렇게 해주세요”보다는 “이 함수의 시간 복잡도를 O(n) → O(log n)으로 개선해주세요”처럼 명확히 요구합니다.
   - 복잡한 작업 분해: 한 번에 많은 일을 시키기보다 세부 단계로 쪼갭니다.
   - 입력/출력 예시 포함: 기대하는 예시를 보여줘야 AI가 잘 따라옵니다.
   - 반복·수정: 첫 답변에 만족 못 하면 질문을 구체화하거나 재구성합니다.
   - 명확성과 일관성 유지: 스타일, 용어, 포맷이 일관되어야 혼란이 없습니다.
   - 역할·페르소나 활용: “시니어 엔지니어 역할을 맡아 코드 리뷰를 해주세요” 등으로 응답 품질을 높입니다.

2. 실전 활용 시나리오

   - 버그 탐지 및 수정
     에러 메시지·잘못된 동작 설명과 함께 최소 재현 가능한 코드 스니펫을 제공하고, “이 함수의 변수 값이 어디서 엉키고 있나요?”처럼 구체적 질문을 던집니다.
   - 코드 리팩토링·최적화
     “리팩토링해주세요” 대신 “가독성을 높이고 복잡도를 줄여주세요”처럼 목표를 명확히 제시하고, “시니어 엔지니어 역할을 맡아 변경 사항을 설명해주세요”라고 요청합니다.
   - 신규 기능 구현
     전체 기능 개요를 Plain English(혹은 자연어)로 설명한 뒤, 하위 작업으로 분해하여 단계별로 묻습니다. 기존 프로젝트의 유사 코드나 아키텍처 스타일을 함께 제공하면 더 정확한 제안이 가능합니다.

3. 주의할 ‘나쁜’ 프롬프트 패턴

   - 구체적이지 않은 요청 (“잘 안 돼요, 고쳐주세요”)
   - 한 번에 너무 많은 작업 요구
   - 성공 기준을 명시하지 않음
   - AI가 묻는 추가 질문을 무시
   - 모호한 지시어 사용 (“이전 출력” 등)

4. 전술적 프롬프트 재작성

   - AI가 다른 문제를 풀고 있거나 에러가 발생했다면, 핵심 불일치 지점(언어, 요구사항 누락 등)을 정확히 짚어서 수정합니다.
   - 복잡한 요청은 다시 쪼개서 단계별로 묻고, 필요 시 채팅 히스토리를 초기화하고 처음부터 재입력합니다.

## 적용 팁

1. 최소 재현 코드 스니펫 제공

   - 실제 코드베이스가 크더라도, 문제를 보여주는 최소한의 코드만 잘라서 전달하세요.
   - 예시:

     ```typescript
     // formatPrice 함수 테스트
     function formatPrice(amount: number): string { … }
     console.log(formatPrice(2.5)); // 기대: '$2.50'
     ```

2. 구체적 성능 목표 명시

   - “이 함수 실행 속도를 50% 줄여주세요”처럼 정량 목표를 달아주세요.

3. 롤(페르소나) 프롬프트 활용

   - “당신은 NestJS 전문가 시니어 개발자입니다. 이 코드를 리뷰해주세요”

4. 단계별 질문 흐름 설계

   - 신규 기능:

     1. 전체 개요 설명
     2. 세부 모듈별 요구사항 요청
     3. 구현된 코드 리뷰 및 테스트 케이스 작성

5. TODO/주석을 인라인 프롬프트로 활용

   - IDE 내에 `// TODO: 사용자 인증 로직 추가` 등으로 표시하고, AI에게 해당 부분만 완성토록 합니다.

6. 반복 대화에서 ‘추가 요구사항 강조’

   - “이전 답변에 타입스크립트 타입 선언이 빠졌습니다. 타입 주석을 포함해 다시 작성해주세요.”

7. 프롬프트 라벨링

   - 여러 질문을 섞어야 할 경우, `[1. 디버그]`, `[2. 리팩토링]`처럼 라벨을 붙여 AI가 혼동하지 않게 유도합니다.
