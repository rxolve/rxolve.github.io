브라우저에서 URL을 인코딩하는 데 사용하는 표준은 RFC 3986과 HTML Form Encoding 두 가지가 주로 사용됩니다.

#### RFC 3986

- encodeURI 함수.
- URL 주소 전체의 일반적인 인코딩 규칙.
- 예약된 문자는 특별한 의미를 가지므로, 필요할 때만 인코딩.

  - :, /, ?, #, [, ], @, !, $, &, ', (, ), \*, +, ,, ;, =

- 예약되지 않은 문자는 그대로 사용.

  - 알파벳 (a-z, A-Z), 숫자 (0-9), -, ., \_, ~는 인코딩 패스.

- 사용 불가능한 문자는 반드시 퍼센트 인코딩으로 변환.
  - 예: 공백(스페이스)은 %20으로 인코딩.
  - 특수 문자(예: @)는 %40으로 변환.

#### HTML Form Encoding

- encodeURIComponent 함수.
- : 폼 데이터 전송 시 사용하는 인코딩 방식.
- RFC 3986과 유사하지만, 공백을 %20으로 인코딩하는 대신 +로 인코딩.
