일반적으로 배열에서 특정 아이템을 제거할 때 `splice`보다는 `filter`가 더 유용합니다.

#### `splice`의 문제점

```typescript
const arr = [1, 2, 3, 4];
arr.splice(1, 1); // 인덱스 1의 요소(2)를 제거
console.log(arr); // [1, 3, 4]
```

- `splice`는 원본 배열을 직접 변경(mutate)하기 때문에 사이드 이펙트가 발생할 수 있습니다.
- 인덱스를 기반으로 제거하기 때문에 요소를 찾는 과정이 따로 필요합니다.
- 반복문 내에서 사용할 경우 인덱스가 변경되어 의도치 않은 동작을 할 수 있습니다.

#### `filter`의 장점

```typescript
const arr = [1, 2, 3, 4];
const newArr = arr.filter((item) => item !== 2);
console.log(newArr); // [1, 3, 4]
```

- `filter`는 원본 배열을 변경하지 않고 새로운 배열을 반환합니다(불변성 유지).
- 특정 조건을 만족하는 요소만 남기므로 가독성이 좋습니다.
- 인덱스를 신경 쓰지 않고 값을 기준으로 필터링할 수 있어 유연합니다.
