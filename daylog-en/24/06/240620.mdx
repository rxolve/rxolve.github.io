- If you rearrange code in an order that's easy to read, you can meet the code in that order.
- When you read code, try to think like a reader.
- Don't give in to the temptation to do other code cleaning at the same time.
  - You can clean up the details you've just found and put them in reading order in the next round of tidying. Don't mix them up.
- What's the best order for code?
- Give the gift of well-ordered code to the next person who reads it.

> Code is written once but read hundreds of times.
> I really connect with the idea of thinking like a reader.
