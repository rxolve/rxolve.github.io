- 얼마전 구현한 멋진 방문자 카운터가 GitHub Pages에서 작동하지 않았습니다.
  - GitHub Pages는 정적 웹 사이트 호스팅 서비스이기 때문에 서버사이드 렌더링(SSR)을 지원하지 않습니다.
- 기존에 사용하던 vercel/postgres 는 무용지물이 되어서 대안을 찾았습니다. 제 선택은 [Supabase](https://supabase.io/) 입니다.

1. 우선 Supabase에 가입하고, 새로운 프로젝트를 생성합니다.
2. 환견변수에 Supabase URL과 API Key를 설정합니다.

   - 프로젝트를 생성하면 확인할 수 있습니다.
   - NEXT_PUBLIC 접두사를 붙여야 클라이언트에서 접근할 수 있습니다.

   ```bash
   NEXT_PUBLIC_SUPABASE_URL=your-supabase-url
   NEXT_PUBLIC_SUPABASE_API_KEY=your-supabase-key
   ```

3. `@supabase/supabase-js` 라이브러리를 설치합니다.

```bash
npm install @supabase/supabase-js
```

4. supabase 클라이언트를 설정합니다. lib/supabase.ts 파일을 생성하고 다음과 같이 작성합니다.

```typescript
import { createClient } from "@supabase/supabase-js";

const supabaseUrl = process.env.NEXT_PUBLIC_SUPABASE_URL;
const supabaseApiKey = process.env.NEXT_PUBLIC_SUPABASE_API_KEY;

if (!supabaseUrl || !supabaseApiKey) {
  throw new Error("Missing env variables for Supabase");
}

export const supabase = createClient(supabaseUrl, supabaseApiKey);
```

5. 방문자 카운터를 구현합니다.
   ```typescript
   const updateVisitorCount = async () => {
     const { data, error } = await supabase
       .from("visitors")
       .select("count")
       .eq("id", 1);
     const visitorCount = data?.length ? data[0].count : 0;
     await supabase
       .from("visitors")
       .update({ count: visitorCount + 1 })
       .eq("id", 1)
       .select();
   };
   ```

- supabase는 vercel/postgres에 비해 더 편리한 UX와 가능을 가지고 있습니다.
  - 서버와 클라이언트에서 supabase client를 통해 데이터베이스에 손쉽게 접근할 수 있습니다.
  - 콘솔에서 데이터베이스를 관리하는 것도 더 편리합니다.
- 둘 다 무료로 사용할 수 있지만, vercel/postgres는 삭제했습니다.
