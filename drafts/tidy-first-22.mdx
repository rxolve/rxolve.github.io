> 정리하며 책의 문장을 요약해서 옮깁니다.

이제는 `소프트웨어 설계의 의미`에 대해 `요소들을 유익하게 관계 맺는 일`이라고 말할 수 있습니다.

고작 세 단어이지만 중요한 개념입니다. 단어 하나하나에 상당한 무게가 담겼다고 할 수 있습니다.

설계란 무엇일까요? 설계는 구성하는 요소들과 그들의 관계, 그리고 그 관계에서 파생되는 이점이 바로 설계입니다.

설계자는 어떤 일을 할까요? 요소들을 유익하게 관계 맺는 일을 합니다.

설계자는 오직 다음과 같은 일만 할 수 있습니다.

- 요소를 만들고 삭제합니다.
- 관계를 만들고 삭제합니다.
- 관계의 이점을 높입니다.

저자가 가장 좋아하는 예시입니다.

- Before:

```
caller()
    return box.width() * box.height()
```

- After:

```
caller()
    return box.area()

Box>>area()
    return width() * height()
```

설계 관점에서 Box.area()라는 새로운 요소를 만들고 caller() 함수와 box 객체 사이의 관계를 조정했습니다.

이제 두 요소는 하나의 함수 호출로 연결됩니다.

시스템의 구조에 대해 말씀드리면 다음과 같습니다.

- 요소 계층 구조
- 요소 사이의 관계
- 이러한 관계가 만들어내는 이점

이제 시스템의 구조와 동작을 좀 더 명확하게 구분할 때입니다.
