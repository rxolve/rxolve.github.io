## 1. 쿠버네티스란?

쿠버네티스는 컨테이너화된 애플리케이션의 배포, 확장 및 관리를 자동화하는 오픈소스 플랫폼입니다. AWS의 ECS나 Elastic Beanstalk과 유사한 역할을 하지만, 더 넓은 생태계와 유연성을 제공합니다.

## 2. AWS 서비스와 쿠버네티스 개념 비교

| 쿠버네티스       | AWS 상응 서비스     | 설명                                             |
| ---------------- | ------------------- | ------------------------------------------------ |
| Pod              | ECS Task            | 가장 작은 배포 단위, 하나 이상의 컨테이너 포함   |
| Node             | EC2 인스턴스        | 애플리케이션을 실행하는 물리적/가상 머신         |
| Cluster          | ECS Cluster         | 노드의 집합, 전체 쿠버네티스 시스템              |
| Deployment       | ECS Service         | 애플리케이션 배포 및 업데이트 관리               |
| Service          | ALB + Target Group  | 애플리케이션에 대한 네트워크 접근 관리           |
| Ingress          | ALB Rules           | 외부에서 클러스터 내 서비스로의 HTTP 라우팅 규칙 |
| ConfigMap/Secret | SSM Parameter Store | 설정 및 비밀 정보 관리                           |

## 3. 쿠버네티스의 장점

1. **높은 이식성**: 멀티 클라우드 및 하이브리드 클라우드 환경 지원
2. **세밀한 리소스 관리**: 컨테이너 레벨의 리소스 할당 및 관리
3. **선언적 구성**: Infrastructure as Code의 철학을 더욱 강력하게 구현
4. **자동 복구 및 자가 치유**: 장애 발생 시 자동으로 복구
5. **롤링 업데이트 및 롤백**: 무중단 배포 및 손쉬운 버전 관리
6. **서비스 디스커버리 및 로드 밸런싱**: 내장된 DNS 기반 서비스 디스커버리
7. **확장성**: 수천 개의 노드와 수만 개의 포드 관리 가능

## 4. AWS에서 쿠버네티스로의 전환을 고려해야 할 때

1. 복잡한 마이크로서비스 아키텍처 (20개 이상의 서비스)
2. 멀티 클라우드 또는 하이브리드 클라우드 전략 수립 시
3. 리소스 사용 최적화가 필요할 때 (월간 클라우드 비용 30% 이상 초과)
4. 더 세밀한 스케일링 제어가 필요할 때 (초 단위 스케일링)
5. 고가용성 요구 증가 (99.99% 이상의 가용성)
6. 개발-테스트-배포 주기 단축 필요 (일 단위 이하 릴리스)
7. AWS 서비스로 충족하기 어려운 특정 요구사항 발생 시

## 5. 쿠버네티스 학습을 위한 첫 걸음

1. Docker 컨테이너화에 대한 이해 심화
2. 쿠버네티스 기본 개념 학습 (Pod, Deployment, Service 등)
3. Minikube나 kind를 이용한 로컬 쿠버네티스 환경 구축
4. AWS EKS (Elastic Kubernetes Service) 살펴보기
5. 헬름(Helm) 차트를 이용한 애플리케이션 배포 연습
6. 쿠버네티스 네트워킹 및 보안 정책 학습
