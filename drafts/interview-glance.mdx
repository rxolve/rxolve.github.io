단일책임의 원칙: 클래스는 하나의 책임만 가져야 한다.
의존성 역전 원칙: 인터페이스로 통신하여 모듈간의 결합도를 낮추는 것
개방폐쇄 원칙: 확장에는 열려있고 수정에는 닫혀있어야 한다.
리스코프 치환 원칙: 자식 클래스는 부모 클래스에서 가능한 행위를 수행할 수 있어야 한다.
인터페이스 분리 원칙: 클라이언트가 사용하지 않는 메서드에 의존하지 않아야 한다.

쿠버네티스: 컨테이너화된 애플리케이션의 배포, 확장 및 관리를 자동화하는 오픈소스 플랫폼. Beanstalk과 유사한 역할을 하지만, 더 넓은 생태계와 유연성을 제공.
카프카: 분산형 메시지 스트리밍 플랫폼. 주로 대용량 트래픽과 분산 처리가 필요한 환경에서 사용.
레디스: 인메모리 데이터 저장소로, 빠른 데이터 조회와 캐시. Redis Streams와 Pub/Sub을 통해 간단한 메시지 큐로도 활용.

제1정규형: 모든 속성의 도메인이 원자값으로만 구성되어 있어야 한다. 각 컬럼이 하나의 값만을 가지도록 합니다.
제2정규형: 부분 함수적 종속성을 제거합니다. 부분 함수종속: A, B -> C 일 때 A -> C
제3정규형: 이행적 함수적 종속성을 제거합니다. 이행적 함수종속: A -> B, B -> C 일 때 A -> C

콜스택: 함수의 호출을 기록하는 자료구조. 함수가 호출될 때마다 스택에 쌓이고, 반환될 때마다 스택에서 제거됩니다.
JS 힙: 동적으로 할당된 메모리가 저장되는 공간. 객체, 배열 등의 데이터가 저장됩니다.
이벤트큐: 이벤트 큐에 쌓인 콜백 함수들은 콜 스택이 비어 있을 때, 즉 현재 실행 중인 동기 작업이 모두 끝났을 때 실행될 준비가 됩니다.

    - 매크로태스크: setTimeout, setInterval과 같은 비동기 함수가 실행될 때 생성되는 태스크
    - 마이크로태스크: Promise와 같은 비동기 함수가 실행될 때 생성되는 태스크

이벤트루프: 콜 스택이 비어있을 때 이벤트 큐에 있는 콜백 함수를 콜 스택으로 가져와 실행합니다.

람다웜업: 1분마다 클라우드와치 이벤트브릿지를 만들어 호출. serverless-plugin-warmup.

bulljs: nodejs에서 사용하는 redis 기반의 큐 시스템. Producers, Consumers, Queues로 구성되어 있습니다.

N+1: 1번의 쿼리를 사용해서 N개의 데이터를 가져올 때, 해당 데이터와 관련된 모든 데이터를 가져오기 위해 추가적으로 N번의 쿼리가 실행되는 문제를 말합니다.
Spring에서 JPA hibernate ORM의 경우 OneToMany의 기본 조회 방식(default fetch type)은 지연 로딩(Lazy Loading)이고, ManyToOne의 기본 조회 방식은 즉시 로딩이므로 N+1 문제를 바로 접할 수 있다고 한다.

반면에, TypeORM은 아래와 같이 어떤 종류의 관계에서도 기본 조회 방식은 이거로딩 이다. 이로 인해 N+1 문제를 피할 수 있다.
